B - Integer Division

問題

X>0X > 0X>0 のときは素直に整数除算 X/10X / 10X/10 を出力する.

X<0X < 0X<0 のとき, X≡0mod 10X \equiv 0 \mod 10X≡0mod10 なら整数除算 X/10X / 10X/10 を出力し, XX % 10 \neq 0X なら X/10−1X / 10 - 1X/10−1 を出力する.

void solve() {
    ll x;
    cin >> x;
    if (x < 0 && x % 10 != 0) {
        cout << x / 10 - 1 << endl;
    } else {
        cout << x / 10 << endl;
    }
}

E - Subtree K-th Max

問題

解説AC

葉から各部分木に含まれる数の上位20個を配列に保存する. このとき, 配列は降順にソートしておく. 各クエリに対してノード VVV の KKK 番目を出力すれば良い.

vvint graph;
vint score;
vvint child;
const int MAX_KTH = 20;

void dfs(int x, int p) {
    child[x].push_back(score[x]);

    for (int nx : graph[x]) {
        if (nx == p) continue;
        dfs(nx, x);
        for (int i = 1; i <= min((int)child[nx].size(), MAX_KTH); i++) {
            child[x].push_back(child[nx][i-1]);
        }
    }

    sort(rall(child[x]));
}

提出コードだと配列に上位20個でなく, 子ノードの数×20+1\text{子ノードの数} \times 20 + 1子ノードの数×20+1 の数が入ってくるが k−dk-dk−d tree の k=20k = 20k=20 みたいなものなので空間計算量も大したことないので問題ない(多分).

直感的には

  • 各ノードの次数が低いとき

    • 「木が高い」となるが, 長さ 202020 の配列が O(N)O(N)O(N) 回くらい伝搬するので計算量は O(20N)O(20N)O(20N) くらい,

  • 各ノードの次数が高いとき

    • 「気が低い」となり, O(N)O(N)O(N) の操作は大した回数にならないだろうと想像できる.

最悪計算量が Nlog⁡20NN\log_{20}NNlog20​N くらいになりそうな気がするが証明はできていない.
