E - Ranges on Tree

問題

入れ子集合モデルを知っていると比較的簡単に実装方針が立ったのではないかと思う.

ref 第5回　SQLで木構造を扱う～入れ子集合モデル　（1）入れ子集合モデルとは何か

まず葉の数字は Li=RiL_i = R_iLi​=Ri​ とするのが最適. そうでなければ無駄に max⁡{Li,Ri}\max \{L_i, R_i\}max{Li​,Ri​} の値が大きくなってしまう.

また

  • Si⊆Sj⇒[Li,Ri]⊆[Lj,Rj]S_i \subseteq S_j \Rightarrow [L_i, R_i] \subseteq [L_j, R_j]Si​⊆Sj​⇒[Li​,Ri​]⊆[Lj​,Rj​]

  • Si∩Sj=∅⇒[Li,Ri]∩[Lj,Rj]=∅S_i \cap S_j = \empty \Rightarrow [L_i, R_i] \cap [L_j, R_j] = \emptySi​∩Sj​=∅⇒[Li​,Ri​]∩[Lj​,Rj​]=∅

より, 全ての葉は違う数字を振るのが最適である. 葉以外のノードでは Li=min⁡全ての子ノードの LL_i = \min{\text{全ての子ノードの } L}Li​=min全ての子ノードの L, Ri=max⁡全ての子ノードの RR_i = \max{\text{全ての子ノードの } R}Ri​=max全ての子ノードの R というのを 葉の方から決めていけば良い.

ここで, 葉に振る数字は DFS での訪れた順に数字を振らなければならない. 葉への数字の振り方がいい加減だと WA になる.

例えば下図のようなときに, 葉ノード 4, 5, 6, 7 に数字を1から連番で振ると図の ようになるが, これでは S2∩S3=∅S_2 \cap S_3 = \emptyS2​∩S3​=∅ なのに [1,3]∩[2,4]≠∅[1, 3] \cap [2, 4] \neq \empty[1,3]∩[2,4]=∅ になってしまう. そのため, 葉への数字の割り振りは DFS で訪れた順でないといけない. 

int n;
vvint graph;
// P: L, R
vector<P> ans;
int num = 1;

void dfs(int x, int p) {
    // leaf
    if (x != 0 && graph[x].size() == 1) {
        ans[x] = {num, num};
        num++;
        return;
    }

    ll mn = INF, mx = -INF;
    for (int nx : graph[x]) {
        if (nx == p) continue;
        dfs(nx, x);
        chmin(mn, ans[nx].first);
        chmax(mx, ans[nx].second);
    }
    ans[x] = {mn, mx};
}

void solve() {
    cin >> n;
    graph.resize(n);
    ans.resize(n, {INF, -INF});

    rep(i,n-1) {
        int u, v;
        cin >> u >> v;
        u--, v--;
        graph[u].push_back(v);
        graph[v].push_back(u);
    }

    dfs(0, -1);

    for (auto [l, r] : ans) {
        cout << l << ' ' << r << endl;
    }
}
