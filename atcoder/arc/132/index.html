A - Permutation Grid

問題

1-index で考え, grid の AAA と呼ぶことにする.

公式解説の通り, RRR, CCC の並び方がともに 1,2,⋯ ,n1, 2, \cdots, n1,2,⋯,n である場合は, 上から rrr 行目, ccc 行目のマスが r+c≥n+1r+c \geq n + 1r+c≥n+1 である場合に黒で, それ以外では白となる.

ここで行 iii, jjj を入れ替えたとき C1,⋯ ,CnC_1, \cdots, C_nC1​,⋯,Cn​ の結果に影響を与えない. 同様に列 iii, jjj を入れ替えたとき R1,⋯ ,RnR_1, \cdots, R_nR1​,⋯,Rn​ の結果に影響を与えない.

RiR_iRi​, CiC_iCi​ が昇順になるように行・列の入れ替えを行ってできる grid を A′A^\primeA′ としたとき AAA の (r,c)(r, c)(r,c) 成分が A′A^\primeA′ のどの成分に対応するか調べることでマスが黒か白か判定できる.

RRR, CCC は 1,⋯ ,n1, \cdots, n1,⋯,n の順列だから, ソートによって rrr 行目は RrR_rRr​ 行目に, ccc 列目は CcC_cCc​ 列目に移動する. よって Rr+Cc≥n+1R_r + C_c \geq n+1Rr​+Cc​≥n+1 のとき黒, それ以外のときは白となる.

B - Shift and Reverse

問題

  • 操作1: 全体をひっくりかえす. つまり, p1,⋯ ,pnp_1, \cdots, p_np1​,⋯,pn​ を pn,⋯ ,p1p_n, \cdots, p_1pn​,⋯,p1​ に並び替える.

  • 操作2: 先頭の項を末尾に移動させる. つまり p1,⋯ ,pnp_1, \cdots, p_np1​,⋯,pn​ を p2,⋯ ,pn,p1p_2, \cdots, p_n, p_1p2​,⋯,pn​,p1​ に並び替える.

とする.

1∼n1 \sim n1∼n に操作1, 操作2 を複数回作用させてできる数列は次の2パターンある.

  • (i) 1の右隣りに2がある. (1が末尾のときは2が先頭にある)

  • (ii) 1の左隣りに2がある. (1が先頭のときは2は末尾にある)

※ 順列 1,⋯ ,n1, \cdots, n1,⋯,n に 操作1, 操作2 を作用させるということは 1,⋯ ,n1, \cdots, n1,⋯,n を円に沿って並べて, 左に1つずらしたり(操作2), 左右反転(操作2)させているだけなので上の (i), (ii) の2通りのパターンしかできない.

参考: 【競プロ実況】ARC132 AとB【かつっぱ】

1のある index を kkk とする. (0-index)

(i) のとき

ex. 5671234

昇順に並べる方法は次の2通りある

  • 1より左にある要素を末尾にすべて移動させるか

    • 操作2 を kkk 回する

  • 操作1をして降順になるまで操作2を行い, 最後にまた操作1をする

    • 4321765 -> 7654321 -> 1234567

    • 操作1を2回

    • 操作2を n−kn-kn−k 回

    • 合計 n−k+2n-k+2n−k+2 回

2通りのうち操作回数が少ないほうが答え. min⁡(k,n−k+2)\min(k, n-k+2)min(k,n−k+2)

(ii) のとき

ex. 4321765

昇順に並べる方法は次の2通り

  • 降順になるまで操作2をしたあとに操作1をする

    • 4321765 -> 7654321 -> 1234567

    • 降順になるまで操作2を行う回数. k+1k+1k+1 回

    • 操作1を行う. 1回

    • 合計 k+2k+2k+2 回

  • 操作1をしたあとに昇順になるまで操作2を繰り返す

    • 5671234 -> 1234567

    • 操作1を行う. 1回

    • 昇順になるまで操作2を行う. n−(k+1)n - (k+1)n−(k+1) 回

    • 合計 n−kn-kn−k 回

2通りのうち操作回数が少ないほうが答え. min⁡(k+2,n−k)\min(k+2, n-k)min(k+2,n−k)
