C - XOR to All

問題

操作は 0 or 1 回しかする必要がない. 2回以上操作してできる数列は1回の操作によってもできる.

まずこれを示す

数列 AAA から AkA_kAk​ を選んで操作してできた数列を BBB とすると Bi=Ai⊕AkB_i = A_i \oplus A_kBi​=Ai​⊕Ak​ である.

同様にして数列 BBB から BlB_lBl​ を選んですさしてできた数列を CCC とすると Ci=Bi⊕BlC_i = B_i \oplus B_lCi​=Bi​⊕Bl​ である. ここで Ci=Bi⊕Bl=(Ai⊕Ak)⊕(Al⊕Ak)=Ai⊕AlC_i = B_i \oplus B_l = (A_i \oplus A_k) \oplus (A_l \oplus A_k) = A_i \oplus A_lCi​=Bi​⊕Bl​=(Ai​⊕Ak​)⊕(Al​⊕Ak​)=Ai​⊕Al​ となるから, 数列 CCC は数列 AAA に AlA_lAl​ を XOR したものと一緒である.

これより2回の操作は必要ないことがわかった.

あとは AAA のどの要素を選んだときに和が最大になるかを全探索すれば良い. 愚直に調べていくと O(N2)O(N^2)O(N2) のオーダーになるので TLE になってしまう. そこで AAA の全ての数に対して kkk bit 目が立っている要素の数を記録する. これを使うと AkA_kAk​ と XOR とった数列の和は

と書ける. ここで rkr_krk​ は kkk bit 目が立っている要素の数.
