001 - Yokan Party（★4）

問題

得られるスコアを二分探索で調べる. 長さ xxx 以上のピースに分割したとき, K+1K+1K+1 個以上ある場合は最低限スコア xxx は達成でき, 逆に K+1K+1K+1 個未満のときはスコア xxx は達成できない. (切りすぎたピースはくっつけて全体で K+1K+1K+1 個にすればよいので判定時には K+1K+1K+1 以上でよい)

判定するところを以下のように書いたら WA となった.

void solve() {
    ll n, l, k;
    cin >> n >> l >> k;
    vector<ll> a(n+2, 0);
    rep(i,n) cin >> a[i+1];
    a[n+1] = l;

    ll ans = 0;
    ll ok = 1, ng = l;
    while (abs(ok - ng) > 1) {
        ll mid = (ok + ng) / 2;
        ll len = 0;
        int cnt = 0;
        rep2(i,1,n+2) {
            len += a[i] - a[i-1];
            if (len >= mid) {
                cnt++;
                len = 0;
            }
        }

        if (cnt > k) {
            ok = mid;
            chmax(ans, ok);
        } else {
            ng = mid;
        }
    }
    cout << ans << endl;
}

chmax(ans, ok) で ans が更新されるからこれで良いと思っていたのだが N=1N = 1N=1, L=2L = 2L=2, K=1K = 1K=1, A1=1A_1 = 1A1​=1 のとき答えは 1 であるが, 上のプログラムだと 0 を返していた. このような条件のときに while の中にそもそも入らないのが原因だった.

そのため ans を使わない以下のように書けば AC になった.

ll ok = 1, ng = l;
while (abs(ok - ng) > 1) {
    ll mid = (ok + ng) / 2;
    ll len = 0;
    int cnt = 0;
    rep2(i,1,n+2) {
        len += a[i] - a[i-1];
        if (len >= mid) {
            cnt++;
            len = 0;
        }
    }

    if (cnt > k) {
        ok = mid;
    } else {
        ng = mid;
    }
}
cout << ok << endl;

003 - Longest Circular Road（★4）

問題

木の直径を求める問題らしい. 木の直径は求め方はどれでもいいので node を一つ選び, そこから各ノードへの最短距離を計算. 一番遠い node を選んでそこからの最短距離をまた計算. 一番距離が遠いところが木の直径になるらしい.

サイクルがあることを考えてしまったがノード数 NNN, 辺の数 N−1N-1N−1 で連結グラフなのでサイクルの心配をする必要はなかった.

木なので最短距離は幅優先探索でも Dijkstra でもどっちでも大丈夫.
